# リファクタリング品質チェックリスト

## 構造

- [ ] SKILL.md が 100行以下に収まっている（目安）
- [ ] frontmatter の description がスキルのトリガー条件を網羅している
- [ ] 前提メモリセクションが必要なスキルに存在する
- [ ] リファレンステーブルが3列構成（ファイル / 内容 / 参照タイミング）になっている
- [ ] ワークフローの Phase/Step から references への参照パスが明確

## 情報の完全性

- [ ] references に移した内容が SKILL.md のリファレンステーブルから参照されている
- [ ] Serena メモリに退避した情報が前提メモリセクションで宣言されている
- [ ] 汎用化で失われた具体例が Serena メモリに保存されている
- [ ] references 内のインライン参照が残っていない（テーブルに集約済み）

## 汎用化（公開用の場合）

- [ ] プロジェクト固有のファイルパスが含まれていない
- [ ] プロジェクト固有のエンティティ名・状態名が含まれていない
- [ ] チケット番号が実在のものではない（`PROJ-001` 等の汎用表記）
- [ ] ユーザー名・個人名が含まれていない
- [ ] 社内ツール固有の名称が含まれていない
- [ ] 汎用的な技術用語（Prisma, DI, OTP 等）は残してよい

## 関連スキルの整合性

- [ ] スキル間の参照関係がある場合、双方に「関連スキル」セクションが存在する（参照する側に `### 参照先`、参照される側に `### 参照元`）
- [ ] 各スキルのサブセクションは関係に応じた構成になっている（参照先のみ / 参照元のみ / 両方）
- [ ] 参照先テーブルの参照タイミングがワークフローの Phase/Step と対応している
- [ ] 参照はファイルパスではなくスキル名で記載されている
