# Agent Skills リファクタリング

カスタムスキルの構造を標準化し、保守性・再利用性を向上させる。

## 鉄則

1. **分析結果を先に提示する** — いきなり変更せず、対象スキルの現状分析と変更計画をユーザーに示す
2. **スキルの動作を壊さない** — 内容の移動であり、情報の削除ではない。references に移した内容が SKILL.md から正しく参照されていることを確認する
3. **プロジェクト固有情報を Serena メモリに退避してから汎用化する** — 先にメモリに書き、次にスキルを汎用化する。逆順だと情報が失われる
4. **ユーザーの承認を待つ** — 各スキルの変更前に計画を提示し、承認後に実行する

## ワークフロー

### Phase 1: 現状分析

1. **全スキルの構造を把握** — `~/.claude/skills/*/SKILL.md` の行数、既存 references の有無を一覧化
2. **リファクタリング対象を特定** — `references/patterns.md` の基準に照らし、改善候補を洗い出す
3. **分析結果をテーブルで提示** — スキル名、現在の行数、改善ポイント、優先度を提示

### Phase 2: 計画提示

4. **スキルごとの変更計画を作成** — 何を references に移すか、何を Serena メモリに移すか、SKILL.md に何を残すか
5. **ユーザーの承認を待つ**

### Phase 3: リファクタリング実行

6. **Serena メモリの作成/更新**（汎用化が含まれる場合）— 先にプロジェクト固有情報を退避
7. **references ファイルの作成** — 分離する内容を references/ に書き出す
8. **SKILL.md の更新** — 分離した内容をリファレンステーブルへの参照に置き換える
9. **チェックリストで検証** — `references/checklist.md` に従い品質を確認

### Phase 4: 公開用の README 更新（該当する場合）

10. **dot_agents リポジトリの README.md を更新** — Serena メモリの依存関係が変わった場合に反映

## リファレンス

| ファイル | 内容 | 参照タイミング |
|---------|------|--------------|
| `references/patterns.md` | リファクタリングパターン集（分離基準、標準構造、汎用化手法） | Phase 1-3: 分析・計画・実行時 |
| `references/checklist.md` | リファクタリング完了前の品質チェックリスト | Phase 3: 実行完了時 |
