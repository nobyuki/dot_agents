# 分割チケットの description 生成ルール

## セクション選択指針

`jira-ticket/references/templates.md` の実装チケットテンプレートをベースに、分割チケットでの必須/任意を定義する。

| セクション | 必須/任意 | 分割チケットでの扱い |
|-----------|----------|-------------------|
| 背景 | 必須 | 元チケットキーと分割経緯を1行で記述 |
| スコープ | 必須 | 本チケットで実施 + スコープ外（他の分割チケットへの相互参照） |
| 完了条件 | 必須 | 他の子チケットに依存しない形で記述（並行着手可能にする） |
| 実装方針 | 任意 | 元チケットの方針から該当部分を引用。方針が自明なら省略可 |
| 検証方法 | 任意 | 検証が非自明な場合のみ追加 |
| 着手前に決めるべき設計論点 | 任意 | 分割で新たに生じた設計判断がある場合のみ追加 |

## 分割固有ルール

1. **背景の書き方** — 元チケットのキーを明記し、分割の経緯を1行で説明
   - 例: 「元チケット PROJ-101 の粒度チェックにより分割。本チケットは API レイヤーを担当」
2. **スコープの相互参照** — 他の分割チケットで対応する内容をスコープ外に明記
3. **完了条件の独立性** — 各子チケットの完了条件は他の子チケットに依存しない形で記述（並行着手可能にする）
4. **元チケットからの継承** — 元チケットの description 全文から、背景・実装方針で再利用可能な部分を引用する（丸コピーではなく、子チケットのスコープに合わせて要約）
