# 評価レポート テンプレートと基準

## 出力先

`/tmp/review-verify-{pr番号}.md` にファイルとして書き出す。
同一 PR に複数レビュアーのレビューがある場合は `/tmp/review-verify-{pr番号}-{レビュアー名}.md` のように区別する。
プロンプト上に markdown を出力するとコードブロックが崩れて markdown 形式が崩れがちなので、必ずファイルに書き出すこと。
書き出し後、ファイルパスをユーザーに伝える。

## テンプレート

```markdown
# PR #{number} レビュー妥当性評価

## 対象

| レビュー | 投稿者 | 種別 |
|---------|--------|------|
| [リンクテキスト](URL) | 投稿者名 | インラインコメント / PR コメント等 |

---

## {レビュー名}

### 各主張の検証

| # | 主張 | 判定 | 根拠 | スコープ判定 |
|---|------|------|------|-------------|
| {reviewer}-R{round}.{number} | 主張の要約 | 判定記号 | 検証根拠（ファイル:行番号を含める） | スコープ内/境界上/スコープ外 |

### 改善余地
（レビューの観点不足や的外れな提案があれば記載。事実誤認は「各主張の検証」テーブル内で ❌ 不正確 として扱う）

---

## 総合評価

| レビュー | 問題発見力 | 事実の正確性 | 付加価値 |
|---------|-----------|-------------|---------|
| レビュー名 | ◎/○/△/× | ◎/○/△/× | 評価 |

### 追加対応の要否
（対応が必要かどうかの結論）
```

## 判定記号

- `✅ 正確` — 主張が事実と完全に一致
- `⚠️ 部分的に正確` / `⚠️ 過剰断定` — 方向性は正しいが範囲や精度に問題
- `❌ 不正確` — 主張が事実と異なる

## 問題発見力の評価基準

- `◎` — 修正前のコードから問題を自力で発見した
- `○` — 既知の問題を正しく認識・整理した
- `△` — 修正後のコードを確認したのみ（問題発見はしていない）
- `×` — 誤った指摘や見落としがある
