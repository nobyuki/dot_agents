# 品質チェックリスト

計画書を完成させる前に確認:

- [ ] 背景セクションで「なぜこの変更が必要か」が明確
- [ ] 影響範囲テーブルで全コンポーネントを網羅
- [ ] 設計詳細のコードスニペットが実在するコードベースの構造に即している
- [ ] Before コードが現行コードと一致している（`find_symbol` で実体を確認。既存ガードの見落としを防ぐ）
- [ ] エラークラスの継承関係を確認済み（例: `TooManyRequestsError extends ApplicationError` → API ハンドラでの捕捉挙動が変わる）
- [ ] ステータス名・定数名が実際の型定義と一致している（型定義ファイルで照合）
- [ ] 既存実装パターンとの整合性を確認済み
- [ ] 後方互換性・マイグレーション戦略を考慮事項に記載
- [ ] 設計詳細の全ての振る舞い（エラーハンドリング、分岐、レート制限等）に対応する実装ステップが存在する（設計→ステップの対応漏れを防ぐ）
- [ ] 実装ステップが作業順序として合理的
- [ ] 画面更新がある場合、手動確認ステップを含んでいる
- [ ] 変更ファイル一覧に漏れがない
- [ ] スコープ外を明示（タスクの肥大化を防止）
- [ ] コミットハッシュが含まれていない
