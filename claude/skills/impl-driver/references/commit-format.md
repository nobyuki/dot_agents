# コミットメッセージのフォーマット

```
<type>(<scope>): <件名（短く、50文字目安）>

<本文: 変更内容の箇条書き>

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

## type の選び方

| type | 用途 |
|------|------|
| `fix` | 不具合修正、異常系の改善 |
| `feat` | 新機能追加 |
| `refactor` | 動作を変えないコード改善 |
| `test` | テストのみの変更 |
| `docs` | ドキュメントのみの変更 |

## scope の例

- チケット番号: `PROJ-101,PROJ-102`
- レイヤー: `domain`, `usecase`, `api`, `ui`

## 注意

- 件名は短く。詳細は本文に書く
- 計画書との差異の説明はコミットメッセージに含めない（計画書を更新する）
- ユーザーのフィードバックがあればそれに従う
