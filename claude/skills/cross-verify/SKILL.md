# AI Output Cross-Verifier

AI 出力に含まれる主張を、現行コードとドキュメントに照らして事実照合する。

## 入力要件

ユーザーから以下を確認してから検証を開始する:

1. **対象テキスト** — 検証対象の AI 出力（ペースト済みであること）
2. **検証対象ブランチ** — どのブランチ/コミット時点で照合するか（省略時は現在の HEAD）
3. **出力粒度** — 全主張を検証 / 重要な主張のみ / 特定セクションのみ（省略時は重要な主張のみ）

## ワークフロー

1. **出力の分類** — レビュー指摘 / 計画書 / 実装提案 / ドキュメント修正案 を判別
2. **検証可能な主張の抽出** — 主観的評価・将来の推測を除外し、コードで裏取り可能な主張を列挙
3. **各主張の検証** — Serena のシンボル検索や Grep で実コードと照合
4. **判定テーブルの出力** — 事実判定と推奨対応を分離して提示

判定記号・検証手順の詳細・分類別の検証ポイントは `references/verification-guide.md` を参照。

## 出力フォーマット

### ヘッダー（必須）

| 項目 | 値 |
|------|-----|
| 検証対象 | （AI 出力の種類と概要） |
| 検証時点 | `branch-name` @ `commit-hash` |
| 出力粒度 | 全主張 / 重要のみ / 特定セクション |

### 判定テーブル

| # | 主張 | 事実判定 | 根拠 | 推奨対応 |
|---|------|----------|------|----------|
| 1 | 主張の要約 | 判定記号 | コード参照付き | 要対応 / 対応不要 / 要議論 |

### 総合所見

検証結果の要約と、対応が必要な項目のリスト。

## 既存スキルとの使い分け

- **GitHub PR レビュー URL** → `verify-review`
- **PR のコードレビュー依頼** → `pr-review`
- **ペーストされた AI 出力の事実照合** → `cross-verify`（本スキル）
- **verify-review レポートに対する AI のメタレビュー** → `cross-verify`（本スキル）。検証結果を verify-review のレポートに反映する
