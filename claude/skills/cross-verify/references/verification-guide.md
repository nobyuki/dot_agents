# 検証ガイド

## 判定記号

| 記号 | 意味 |
|------|------|
| ✅ 正確 | 主張が事実と完全に一致 |
| ⚠️ 部分的 | 方向性は正しいが範囲や精度に問題 |
| ⚠️ 過剰断定 | 検証範囲を超えた断定 |
| ❌ 不正確 | 主張が事実と異なる |
| ➖ 検証不能 | 現在のコードベースでは判断できない |

## 推奨対応の判断基準

| 対応 | 基準 |
|------|------|
| 要対応 | 事実判定が正確かつ、修正しないと不具合・不整合が生じる |
| 対応不要 | 正確だが既に対応済み、または設計上意図的な実装 |
| 要議論 | 正確性は確認できたが、対応方針にトレードオフがある |

## Phase 1: 主張の抽出

ペーストされたテキストから検証可能な主張を列挙する。

### 抽出すべき主張の種類

| カテゴリ | 例 |
|---------|-----|
| ファイル・行番号参照 | 「L42 に未使用の import がある」 |
| シンボル・型の参照 | 「SignUpFlowStatus に Cancelled が定義されている」 |
| ロジックの主張 | 「この分岐は到達不能」 |
| 網羅性の主張 | 「全ファイルで統一されている」 |
| 設計の主張 | 「DI コンテナの変更が必要」 |
| 既存コードとの整合性 | 「現行の実装では〜になっている」 |

### 抽出しない主張

- 主観的な品質評価（「コードが読みにくい」等）
- 根拠なしの将来推測（「将来的に問題になる」等）
  - ただし具体的な条件付き（「N件超えると性能劣化」等）なら検証対象

## Phase 2: 検証の実行

### 効率的な検証手段の選択

1. **シンボル名が分かる場合** → Serena の `find_symbol`（include_body=true）
2. **ファイルパスが分かる場合** → ファイルの該当箇所を読む
3. **パターン検索が必要な場合** → Grep または `search_for_pattern`
4. **参照関係の確認** → `find_referencing_symbols`
5. **型定義の確認** → `find_symbol` で型を検索

### 検証の注意点

- **時点差**: AI 出力生成後にコードが変わっている可能性がある。ヘッダーの検証時点と AI 出力の生成時点にズレがあれば、その旨を明記する
- **断定表現**: 「すべて」「常に」「一切ない」は特に厳密に検証する
- **インターフェース越しの主張**: 実装クラスまで読んで検証する
- **網羅性**: 「正しい」だけでなく「他に同種の問題がないか」も確認する

## Phase 3: 総合評価

検証結果を踏まえて以下を判断する:

1. **要対応の指摘** — 正確かつ対応が必要な指摘のリスト
2. **AI 出力の信頼性** — 正確な主張の割合、過剰断定の有無
3. **見落とし** — AI が触れていないが検証過程で発見した関連問題

## メタレビュー指摘の扱い（レポート自体への指摘）

AI ツールがレビューレポート自体の品質（行番号ズレ、存在しない文書参照、セクション間矛盾）を指摘する場合がある。

**対応方針:**
- 指摘が正確なら内部でレポートに反映する
- PR 作者向けの最終レポートにはメタレビュー指摘を残さない（ノイズになる）
- クロス検証セクションにはコードに対する指摘のみ残す

**典型的なメタレビュー指摘:**
- 行番号の更新漏れ（本文は修正済みだがテーブルに残存）
- リポジトリに存在しないスキル内部ファイルを根拠として引用
- デグレチェックの「問題なし」が自身の Finding と矛盾
- 既存パターンの説明が実コードより過剰

## 出力分類別の検証ポイント

### レビュー指摘

- GitHub URL があるなら verify-review にリダイレクトすることを提案
- 指摘対象のファイル・行番号を特定してコードと照合
- 修正前と修正後のどちらを見ているか判別する

### 計画書ドラフト

- 変更対象のファイル・シンボルが実在するか
- Before/After のコードスニペットが現行コードと一致するか
- 依存関係（DI コンテナ、テストファイル等）の変更漏れがないか
- 変更ステップの順序に矛盾がないか

### 実装提案

- 参照している API・型定義が正確か
- 既存パターンとの一貫性
- 副作用（テスト破壊、DI 引数変更等）への言及の有無

### ドキュメント修正案

- 用語・状態名が実装と一致するか
- 網羅性（修正すべき箇所が他にないか）
