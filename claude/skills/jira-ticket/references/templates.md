# description テンプレート

チケット種別ごとの description テンプレート。種別の判断基準は `workflow.md` Phase 1 を参照。

## 実装チケット

```markdown
## 背景
（何が起きたか / なぜこのチケットが必要か）

## スコープ
### 本チケットで実施
（番号付きリストで列挙）

### スコープ外（将来対応）
（明示的に除外する項目）

## 実装方針
（技術的な方針・設定内容）

## 完了条件
（番号付きリストで検証可能な条件を列挙）

## 検証方法
（完了条件の検証手順が非自明な場合に記載）
```

### 設計判断が未確定な場合

実装方針に未決定の選択肢がある場合、「実装方針」セクションに選択肢を並記するのではなく、「背景」と「スコープ」の間に「着手前に決めるべき設計論点」セクションを追加する。

```markdown
## 着手前に決めるべき設計論点

以下の論点は実装着手前にチーム内で合意すること。

1. **（論点名）**
   - （判断の選択肢や考慮事項を箇条書きで）
```

このセクションを追加した場合、以下も連動して調整する:
- **スコープ**: 先頭に「上記設計論点の合意」を入れる
- **完了条件**: 先頭に「設計論点の合意がチケットコメントに記録されている」を入れ、以降の条件は「合意した方針に基づき〜」のように合意結果に依存する形で書く（「適切な〜」のような曖昧表現を避ける）
- **実装方針**: 省略するか、設計論点の合意後に確定する旨を記載

## 方針策定チケット

方針・ルール・ガイドラインの策定が目的で、コード変更を伴わないチケット。成果物はドキュメント（`.claude/rules`、CLAUDE.md、ガイドライン等）。

```markdown
## 背景
（なぜこの方針が必要か / 何が未明文化か）

## スコープ
### 本チケットで実施
（方針文書の作成、既存違反箇所の洗い出し、後続チケット化ルールの決定など）

### スコープ外（後続チケットで対応）
（既存コードの修正、テスト追加、エラー型統一等の実装作業）

## 方針内容
（策定する方針の具体的な内容）

## 既知の違反箇所（参考）
（方針に照らして是正が必要な箇所の代表例。修正対象としてではなく参考情報として記載）

## 完了条件
（方針文書の作成、レビュー承認等。検証可能な条件で記載）
```

### 方針策定チケットの注意点

- **実装詳細を混ぜない**: 方針チケットにコード修正・テスト追加・エラー型選定等の実装作業を含めると、レビューが実装の穴探しに変わり、方針合意が進まなくなる
- **違反箇所は「参考」に留める**: 既知の違反箇所は参考情報として記載し、修正対象としては書かない。是正は後続チケットで行う
- **完了条件は文書と合意**: 「方針文書が作成されている」「PR レビュー承認により合意されている」のようにドキュメント成果物で判定する
- **数値は断定しない**: 「43ファイルで使用」のような具体数は計測タイミングでずれるため、「多数のファイルで使用」に寄せる（落とし穴「背景の数値データの脆さ」参照）

## セクション選択の指針

### 実装チケット

| セクション | 必須/任意 |
|-----------|----------|
| 背景 | 必須 |
| 着手前に決めるべき設計論点 | △ 未決定の設計判断がある場合に追加 |
| スコープ | 必須 |
| 実装方針 | △ 方針が自明なら省略可 |
| 完了条件 | 必須 |
| 検証方法 | △ 自明なら省略可 |

### 方針策定チケット

| セクション | 必須/任意 |
|-----------|----------|
| 背景 | 必須 |
| スコープ | 必須 |
| 方針内容 | 必須 |
| 既知の違反箇所（参考） | △ 違反箇所が既知の場合に追加 |
| 完了条件 | 必須 |

## issue type の選び方

| type | 用途 |
|------|------|
| タスク | 機能追加、設定変更、CI/CD 改善など |
| バグ | 不具合修正 |
| ストーリー | ユーザー目標として表現される機能 |
| スパイク | 調査・検証タスク |

ユーザーが指定しない場合はデフォルトで「タスク」を使用する。
