# description テンプレート

チケットの性質に応じてテンプレートを使い分ける。

## 実装チケット

```markdown
## 背景
（何が起きたか / なぜこのチケットが必要か）

## スコープ
### 本チケットで実施
（番号付きリストで列挙）

### スコープ外（将来対応）
（明示的に除外する項目）

## 実装方針
（技術的な方針・設定内容）

## 完了条件
（番号付きリストで検証可能な条件を列挙）

## 検証方法
（完了条件の検証手順が非自明な場合に記載）
```

## 合意形成チケット

実装ではなく意思決定が目的のチケット（方針決定、設計判断など）。

```markdown
## 背景
（現状と、方針決定が必要な理由）

## 論点
（番号付きリストで判断すべきポイントを列挙）

## 選択肢
### A: （選択肢名）
（説明・実装イメージ）

### B: （選択肢名）
（説明・実装イメージ）

### C: （選択肢名）
（説明・実装イメージ）

## 完了条件
（採用案・選定理由の記録、後続チケットの起票など）
```

## セクション選択の指針

| セクション | 実装チケット | 合意形成チケット |
|-----------|------------|----------------|
| 背景 | 必須 | 必須 |
| スコープ | 必須 | - |
| 論点 | - | 必須 |
| 選択肢 | - | 必須 |
| 実装方針 | △ 方針が自明なら省略可 | - |
| 完了条件 | 必須 | 必須 |
| 検証方法 | △ 自明なら省略可 | - |

## issue type の選び方

| type | 用途 |
|------|------|
| タスク | 機能追加、設定変更、CI/CD 改善など |
| バグ | 不具合修正 |
| ストーリー | ユーザー目標として表現される機能 |
| スパイク | 調査・検証タスク |

ユーザーが指定しない場合はデフォルトで「タスク」を使用する。
