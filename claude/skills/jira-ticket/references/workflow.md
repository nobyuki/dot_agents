# JIRA チケット作成ワークフロー詳細

## Phase 1: ヒアリング・情報整理

1. **ユーザーの要件を整理** — 何を解決したいか、背景・原因・対応方針を把握
2. **不明点があれば質問** — スコープ、優先度、issue type 等が不明なら確認。issue type は未指定ならデフォルトで「タスク」を使用する

## Phase 2: ドラフト作成

3. **タイトルと description の案を提示** — `references/templates.md` のテンプレートに従い、markdown コードブロックで表示
4. **ユーザーのフィードバックを待つ** — 修正指示・レビューコメントがあれば Phase 3 へ

## Phase 3: レビュー反映（繰り返し）

5. **レビューコメントの妥当性を検証** — 現在のドラフトに照合し、判定テーブルを提示

   | # | 指摘 | 判定 | 根拠 |
   |---|------|------|------|
   | 1 | 指摘内容 | ✅ 対応済み / ✅ 妥当 / ⚠️ 部分的 / ❌ 不要 | 根拠 |

6. **妥当な指摘のみ反映してドラフトを更新** — 更新後の全文を再表示
7. **Phase 2-3 をユーザーが満足するまで繰り返す**

## Phase 4: JIRA 登録

8. **ユーザーの「作成して」「登録して」を待つ** — 承認前に絶対に API を呼ばない
9. **プロジェクト情報を取得** — `getAccessibleAtlassianResources` で cloudId、`getJiraProjectIssueTypesMetadata` で issue type を取得
10. **チケットを作成** — `createJiraIssue` で登録し、チケットキーと URL を返す
